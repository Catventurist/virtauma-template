## How To Contribute

### Fix Issues and Help Out

Conversations can be had in the [issues](https://github.com/catventurist/kaannos/issues) and [discussions](https://github.com/catventurist/kaannos/discussions) boards.

### Creating an Issue

- Reporting bugs: Check out the Nuxt [guide](https://nuxt.com/docs/4.x/community/reporting-bugs) for some things to do before opening an issue.
- Feature requests: See the **Ideas** section

### Send a Pull Request

Check whether there's an issue that describes it, as it's possible there is a documentation issue or provide some context, that would be helpful to know.

Open a feature request issue first to discuss with the maintainers whether the feature is desired.

For bigger changes to Nuxt itself, we recommend that you first create a Nuxt module.

Possibility to use [Conventional Commits](https://www.conventionalcommits.org/) for commit messages, which allows a changelog to be auto-generated based on the commits.

- A [guide](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/creating-a-pull-request) for making a pull request
- Title should follow the [Commit Convention](https://nuxt.com/docs/4.x/community/contribution#commit-conventions).

Once you've made a pull request, we'll do our best to review it promptly.

### AI-Assisted Contributions

**Avoid** using algorithmically hallucinated code _slop_!

- Never let an LLM speak for you
- Never let an LLM think for you

Our aim is ensuring quality and maintaining the joy of collaborating and communicating with real people.

[ESLint](https://github.com/nuxt/eslint) is good for both linting and formatting with `@nuxt/eslint`.
