## General Structure

Virtauma is a **Nuxt layer** that extends a base Nuxt application with documentation features. This enables full Nuxt support.

When creating a new project using `npm create nuxt@latest`, you get:

```
├── content/             # Markdown content
│   ├── en/index.md      # Homepage in English
│   │    └── docs/       # Documentation in English
│   ├── fi/index.md      # Homepage in Finnish
│        └── docs/       # Documentation in Finnish
├── public/              # Static assets, images
└── package.json         # Dependencies and scripts
```

You can use any Nuxt feature or file:

```
├── app.config.ts        # Application configuration
├── nuxt.config.ts       # Nuxt configuration (add modules, components, etc.)
├── app/                 # Application directory
│   ├── components/      # Components (add your own)
│   ├── layouts/         # Layouts (add your own)
│   └── pages/           # Pages (add your own)
└── server/              # Server code (add your own)
```

### `content/` Directory
This is where you add Markdown pages. Routes are generated automatically based on the file structure.

**Single-language structure:**
```
content/
├── index.md
├── getting-started.md
└── guide/
    ├── introduction.md
    └── configuration.md
```

**Multi-language structure (using i18n):**
```
content/
├── en/
│   ├── index.md
│   └── guide/
│       └── introduction.md
└── fi/
    ├── index.md
    └── guide/
        └── introduction.md
```

::tip
More information about **i18n** translation features can be found in the [Translations](/en/docs/getting-started/localization) section.
::

### `public/` Directory

Files in the `public/` directory are served from the root and are not processed during the build. This is where images, icons, and other static files live.

### `package.json`

This file contains all the application's dependencies and scripts.

Virtauma's `package.json` is very minimal and looks like this:

```json [package.json]
{
  "name": "docs",
  "scripts": {
    "build": "nuxt build",
    "dev": "nuxt dev",
  },
  "dependencies": {
    "better-sqlite3": "^12.2.0",
    "nuxt": "^4.0.0"
  }
}
```

### `nuxt.config.ts`

**This file is not required in Virtauma.**

You can use additional modules in the Nuxt configuration file:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  extends: ['@nuxtjs/i18n']
})
```

### `app.config.ts`

**This file is not required in Virtauma.**

This is where the [Virtauma configuration](/en/docs/getting-started/usage) lives, which can be customized to fit your theme, handle SEO, and add links.

## Full Nuxt Project Potential

Since Virtauma is a Nuxt layer, you can use **any** Nuxt feature:

```txt
my-docs/
├── app/                 # Application directory (optional)
│   ├── components/      # Custom Vue components
│   ├── layouts/         # Custom layouts
│   ├── pages/           # Custom Vue pages (outside `/content`)
│   ├── composables/     # Vue composables
│   └── middleware/      # Route middleware
├── server/              # Server code
│   └── api/             # API routes
├── plugins/             # Nuxt plugins
├── middleware/          # Global middleware
└── modules/             # Custom Nuxt modules
```

::tip{to="/en/docs/getting-started/usage"}
With this layer-based approach, you can leverage the entire Nuxt ecosystem while staying focused on creating content.
::
