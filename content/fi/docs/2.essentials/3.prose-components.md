### Haitari

Käytä `accordion` ja `accordion-item` luodaksesi kokoontaitettavia sisältöosioita. Haitari on kätevä UKK:n, laajennettavien yksityiskohtien ja ryhmitellyn tiedon esittelemiseen interaktiivisesti.

```md
::accordion

:::accordion-item{label="Miksi päivittää?" icon="i-lucide-circle-help"}
Muutoksia syntyy, kuten uudet komponenttirakenteet, teemat ja kehittyneet TyyppiScript-määritteet. Suosittelemme asteittaista päivitystä kehitysympäristössä testaillen.
:::

:::accordion-item{label="Yhteensopivuus?" icon="i-lucide-circle-help"}
Täysin Vue-yhteensopiva! Voit alkuun lueskella [asennusopasta](/fi/docs/getting-started/installation).
:::

::
```

### Leima

Käytä leimoja tilojen tai merkintöjen näyttämiseen. Niillä voi korostaa versionumeroita tai kategorioita sisällössä.

```md
::badge
**v4.0.0**
::
```


### Huomioteksti

Käytä huomiotekstiä korostaaksesi oleellista tietoa. Ne voivat olla muistioita, vinkkejä, varoituksia tai huomautuksia.

Mukauta käyttäen`icon` ja `color`-ominaisuuksia tai `note`, `tip`, `warning`, `caution`-lyhenteitä esimääritellyillä tyyleillä.


```md
::callout
Tämä on `callout` täydellä **markdown**-tuella.
::
```

```md
::note
Lisätietoja.
::

::tip
Erikoinen ehdotus.
::

::warning
Toiminto voi tuottaa odottamattomia tuloksia.
::

::caution
Toimintoa ei voi peruuttaa.
::
```

### Kortti

Käytä `card` korostaaksesi sisältölohkoja. Käteviä ominaisuuksien, resurssien tai oleellisen tiedon esittämiseen omanlaatuisten ja interaktiivisten säiliöiden avulla.

Muokkaa `title`, `icon`, ja `color`-ominaisuuksia. Kortit toimivat myös linkkeinä `<NuxtLink>`-ominaisuuksineen navigoinnissa.

```md
::card
---
title: Kojelauta
icon: i-simple-icons-github
to: https://github.com/nuxt-ui-templates/dashboard
target: _blank
---
Kojelauta usean sarakkeen ulkoasulla.
::
```

### Kokoontaitto

`collapsible` piilottaaksesi tai näyttääksesi sisällön osioita. `collapsible` sopii lisätietoihin, teknisiin erittelyihin tai harvemmin tarvittavaan tietoon.

```md
::collapsible

|Ominaisuus| Oletus    | Tyyppi                   |
|----------|-----------|--------------------------|
| `nimi`   |           | `string`{lang="ts-type"} |
| `koko`   | `md`      | `string`{lang="ts-type"} |
| `väri`   | `neutral` | `string`{lang="ts-type"} |

::
```


### Kentät

Käytä `field` kuvataksesi tiettyä alaa, ominaisuutta tai parametriä. `field`-komponentit sopii hyvin API-parametrien, komponenttiominaisuuksien tai asetusvalintojen dokumentointiin.

```md
::field{name="name" type="string" required}
`description` voidaan asettaa oletuspaikan ominaisuudeksi täydellä **markdown**-tuella.
::
```


### Alaryhmä

Käytä `field-group` ryhmittääksesi kentät yhteen listamuotoon. `field-group` helpottaa dokumentaation hallintaa ja rakentamista useilla oleellisilla kentillä tai ominaisuuksilla.

```md
::field-group
  ::field{name="analytics" type="boolean"}
    Oletus: `false` - Lisää analyysit projektiin (tulossa).
  ::

  ::field{name="blob" type="boolean"}
    Oletus: `false` - Lisää blob-tallennustila staattisille tiedostoille, kuten kuville ja videoille.
  ::

  ::field{name="cache" type="boolean"}
    Oletus: `false` - Lisää välimuisti-tallennustila palvelimen reiteille tai Nitron `cachedEventHandler` ja `cachedFunction`-funktioille.
  ::

  ::field{name="database" type="boolean"}
    Oletus: `false` - Lisää SQL-tietokanta tallentaaksesi ohjelmiston tiedot.
  ::
::
```


### Kuvakkeet

Käytä `icon` lisätäksesi kuvakkeita eri kirjastoista. Ne tarjoavat visuaalisia vihjeitä sekä parantavat dokumentaation käyttöliittymää.

```md
:kbd{value="meta"} :kbd{value="K"}
```


### Välilehdet

Käytä `tabs` ja `tabs-item` järjestääksesi sisältöä välilehtiin. Ne ovat päteviä sisällön jakamiseen loogisiin lokeroihin parantaen löydettävyyttä sekä järjestelmällisyyttä.

````md
::tabs

:::tabs-item{label="Koodi" icon="i-lucide-code"}

```md
::callout
Olemme yhtä.
::
```

:::

:::tabs-item{label="Esikatselu" icon="i-lucide-eye"}

::::callout
Olemme yhtä.
::::

:::

::
````


### Askeleet

Käytä `steps` luodaksesi vaiheittaiset opasteet dokumentin otsikoista. `steps`-komponentti numeroi otsikot automaattisesti tehden opastukset prosesseille ja oppaille.

Aseta `level`-ominaisuus määrittääksesi otsikon tason vaiheittaisessa oppaassa.

```md
::steps{level="4"}

#### Lisää Nuxt-moduuli tiedostoon `nuxt.config.ts`{lang="ts-type"}

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['@nuxt/ui']
})
```

#### Tuo Tailwind CSS ja Nuxt UI CSS

```css [assets/css/main.css]
@import "tailwindcss";
@import "@nuxt/ui";
```
