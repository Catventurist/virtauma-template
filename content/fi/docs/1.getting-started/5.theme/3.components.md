## Tailwind-muuttujat

Virtauma-komponenttien tyylit perustuvat [Tailwind Variants](https://www.tailwind-variants.org/) API:n, joka tarjoaa tehokkaan tavan luoda muuttujia ja hallita komponenttien tyylejä.

### Osiot

Komponenteissa voi olla useita `slots`-osioita, joista jokainen edustaa erityistä HTML-elementtiä komponentissa. Näiden osioiden ansiosta sisällön lisääminen ja tyylittely on sujuvaa.

Esimerkkinä [Card]()-korttikomponentti useammalla osiolla:

::code-group

```ts [src/theme/card.ts]
export default {
  slots: {
    root: 'bg-default ring ring-default divide-y divide-default rounded-lg',
    header: 'p-4 sm:px-6',
    body: 'p-4 sm:p-6',
    footer: 'p-4 sm:px-6'
  }
}
```

```vue [src/runtime/components/Card.vue]
<template>
  <div :class="ui.root({ class: [props.ui?.root, props.class] })">
    <div :class="ui.header({ class: props.ui?.header })">
      <slot name="header" />
    </div>

    <div :class="ui.body({ class: props.ui?.body })">
      <slot />
    </div>

    <div :class="ui.footer({ class: props.ui?.footer })">
      <slot name="footer" />
    </div>
  </div>
</template>
```

::

Joissain komponenteissa ei ole osioita ja ne koostuvat yhdestä juurielemetistä. Tässä tapauksessa vain `base`-osio on määritelty, kuten [Container]()-säiliökomponentissa:

::code-group

```ts [src/theme/container.ts]
export default {
  base: 'max-w-(--ui-container) mx-auto px-4 sm:px-6 lg:px-8'
}
```

```vue [src/runtime/components/Container.vue]
<template>
  <div :class="container({ class: props.class })">
    <slot />
  </div>
</template>
```

::

::warning
Komponentit ilman osioita eivät sisällä [`ui`-ominaisuutta](#ui-prop), vain [`class`-ominaisuuden](#class-prop) on saatavilla tyylien korvaamisessa.
::

### Muuttujat

Komponentit tukevat `variants`-muuttujia, joiden avulla voidaan suoraan määritellä eri `slots`-osioiden tyylejä perustuen komponentin ominaisuuksiin.

Esimerkiksi [Avatar]()-komponentissa käytetään `size`-kokomuuttujaa ulkoasun määrittelemiseen:

```ts [src/theme/avatar.ts] {6-18}
export default {
  slots: {
    root: 'inline-flex items-center justify-center shrink-0 select-none overflow-hidden rounded-full align-middle bg-elevated',
    image: 'h-full w-full rounded-[inherit] object-cover'
  },
  variants: {
    size: {
      sm: {
        root: 'size-7 text-sm'
      },
      md: {
        root: 'size-8 text-base'
      },
      lg: {
        root: 'size-9 text-lg'
      }
    }
  },
  defaultVariants: {
    size: 'md'
  }
}
```

Näin ollen `size`-ominaisuus soveltaa vastaavat tyylit `root`-osioon.

### Oletusmuuttujat

`defaultVariants`-ominaisuus asettaa oletusarvon jokaiselle muuttujalle, kun ominaisuuksia ei ole lisättynä.

Esimerkiksi [Avatar]()-komponentissa on oletuskokona `md`:

```ts [src/theme/avatar.ts] {19-21}
export default {
  slots: {
    root: 'inline-flex items-center justify-center shrink-0 select-none overflow-hidden rounded-full align-middle bg-elevated',
    image: 'size-full rounded-[inherit] object-cover'
  },
  variants: {
    size: {
      sm: {
        root: 'size-7 text-sm'
      },
      md: {
        root: 'size-8 text-base'
      },
      lg: {
        root: 'size-9 text-lg'
      }
    }
  },
  defaultVariants: {
    size: 'md'
  }
}
```

:::tip{to="/fi/docs/getting-started/installation"}

Voit käyttää `theme.defaultVariants`-valintaa `nuxt.config.ts`-tiedostossa ohittaaksesi oletusarvot `size` ja `color` kaikissa komponenteissa kertaheitolla.
:::

### Yhdistelmämuuttujat

Joissain komponenteissa käytetään `compoundVariants`-ominaisuutta luokkiin, kun useat muuttujien ehdot täyttyvät samanaikaisesti.

Esimerkiksi [Button]()-painikekomponentti käyttää `compoundVariants`-ominaisuutta asettaen luokkia tietyille `color` ja `variant`-yhdistelmille:

```ts [src/theme/button.ts] {27-31}
import type { ModuleOptions } from '../module'

export default (options: Required<ModuleOptions>) => ({
  slots: {
    base: ['rounded-md font-medium inline-flex items-center disabled:cursor-not-allowed aria-disabled:cursor-not-allowed disabled:opacity-75 aria-disabled:opacity-75', options.theme.transitions && 'transition-colors']
  },
  variants: {
    color: {
      ...Object.fromEntries((options.theme.colors || []).map((color: string) => [color, ''])),
      neutral: ''
    },
    variant: {
      solid: '',
      outline: '',
      soft: '',
      subtle: '',
      ghost: '',
      link: ''
    }
  },
  compoundVariants: [
    ...(options.theme.colors || []).map((color: string) => ({
      color,
      variant: 'outline',
      class: `ring ring-inset ring-${color}/50 text-${color} hover:bg-${color}/10 active:bg-${color}/10 disabled:bg-transparent aria-disabled:bg-transparent dark:disabled:bg-transparent dark:aria-disabled:bg-transparent focus:outline-none focus-visible:ring-2 focus-visible:ring-${color}`
    })),
    {
      color: 'neutral',
      variant: 'outline',
      class: 'ring ring-inset ring-accented text-default bg-default hover:bg-elevated active:bg-elevated disabled:bg-default aria-disabled:bg-default focus:outline-none focus-visible:ring-2 focus-visible:ring-inverted'
    }
  ],
  defaultVariants: {
    color: 'primary',
    variant: 'solid'
  }
})
```

## Muokkaa teemaa

On useita tapoja muokata Virtauma-komponenttien ulkoasua. Se onnistuu kaikille komponenteille kerrallaan tai yksittäisissä tiedostoissa.

::note
Tailwind Variants-muuttujat käyttää [`tailwind-merge`](https://github.com/dcastil/tailwind-merge)ä oletuksena yhdistäen luokat, jolloin ei tarvitse huolehtia niiden ristiriidoista.
::

::tip
Voit katsella teemaa eri komponenteissa kahdella tavalla:

- Lue `Teema`-osio eri komponenttien kohdalla.
- Katsele lähdekoodia suoraan Github-kirjastossa [`src/theme`](https://github.com/nuxt/ui/tree/v4/src/theme).
::

### Yleisasetukset

Voit ohittaa komponenttien teeman yleistasolla `app.config.ts`-tiedostossa käyttäen teeman rakennetta objektina.
Voit muokata [`slots`](#slots), [`variants`](#variants), [`compoundVariants`](#compound-variants) ja [`defaultVariants`](#default-variants) muuttaaksesi komponentin oletusteemaa:

```ts [app.config.ts]
export default defineAppConfig({
  ui: {
    button: {
      slots: {
        base: 'font-bold'
      },
      variants: {
        size: {
          md: {
            leadingIcon: 'size-4'
          }
        }
      },
      compoundVariants: [{
        color: 'neutral',
        variant: 'outline',
        class: 'ring-default hover:bg-accented'
      }],
      defaultVariants: {
        color: 'neutral',
        variant: 'outline'
      }
    }
  }
})
```

::note
Tässä esimerkissä `font-bold` ohittaa `font-medium` kaikissa painikkeissa, `size-4` ohittaa `size-5`-luokan kuvakkeessa, kun  `size="md"` ja `ring-default hover:bg-accented` ohittaa `ring-accented hover:bg-elevated`, kun `color="neutral"` ja `variant="outline"`. Painikkeista tulee `color="neutral"` ja `variant="outline"`
::

### `ui`-ominaisuus
Voit ohittaa myös komponentin **slots**-osiot käyttämällä `ui`-ominaisuutta. Tämä on priorisoitu yleisasetusten yli ja ratkaisee `variants`.

### `class`-ominaisuus

`class`-luokkaominaisuus mahdollistaa `root` ja `base`-osioiden korvaamisen. Tämä on priorisoitu yleisasetusten yli ja ratkaisee `variants`.

::note
Esimerkiksi `font-bold`-luokka voi ohittaa `font-medium`-oletusluokan.
::
