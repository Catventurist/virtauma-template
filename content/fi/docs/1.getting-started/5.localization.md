## Perusasetukset

Aluksi määritetään projektin `locales` ja `defaultLocale` **nuxt.config.ts**-tiedostoon.

Esimerkissä kielet määritetään seuraavilla ominaisuuksilla:

- `code`: vaaditaan, koodia käytetään Nuxt I18n:ssä ja toimii kielen tunnisteena.
- `name`: kielen nimi. Tämä on käyttäjäystävällinen keino tunnistaa kieli.
- `file`: sanojen ja lauseiden käännöstiedosto objektina.

`defaultLocale` tulee olla jonkin määritellyn kielen `code`. Vaihtoehtoinen, mutta sitä suositellaan käyttämään varmistuksena, kun selataan olemattomalle reitille.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['@nuxtjs/i18n'],
  i18n: {
    defaultLocale: 'en',
    locales: [
      { code: 'en', name: 'English', file: 'en.yml' },
      { code: 'f', name: 'Suomi', file: 'fi.yml' }
    ]
  }
})
```

Yleensä projektissa on vähintään yksi `file`-tiedosto jokaista kieltä kohtaan, josta löytyy käännökset objektimuodossa.

Nuxt I18n sisältää (muokattavan) tiedostorakenteen, josta kielitiedostot haetaan. Ne tulisi luoda oletuksena `<rootDir>/i18n/locales`-kansioon.

::code-group

```json [i18n/locales/en.yml]
{
  "welcome": "Welcome"
}
```

```yml [i18n/locales/fi.json]
{
  "welcome": "Tervetuloa"
}
```

::

Tähän voidaan lisätä kielen vaihtaja ja lisätä ensimmäinen käännöslause:

```vue [pages/index.vue]
<script setup>
const { locales, setLocale } = useI18n()
</script>

<template>
  <div>
    <button v-for="locale in locales" @click="setLocale(locale.code)">
      {{ locale.name }}
    </button>
    <h1>{{ $t('welcome') }}</h1>
  </div>
</template>
```

Käyttäen määriteltyjä kieliä teimme kielenvaihtajan. Klikkaamalla `<button>`-elementtiä voit vaihtaa Englannin ja Suomen välillä, sekä "tervetuloa"-teksti ja URL vaihtuvat kielen mukaan.

Tässä on perusta, josta lähteä rakentamaan uusia ohjelmia!

## Automaattiset tuonnit

Jotkut composable-funktiot, kuten `useI18n`, [tuodaan automaattisesti Nuxt:n toimesta](https://nuxt.com/docs/guide/concepts/auto-imports#auto-imports).
Jos olet poistanut `autoImports`-asetuksen käytöstä, sinun täytyy tuoda ne `#imports`-moduulista seuraavasti:

```vue
<script setup>
import { useI18n, useLocalePath } from '#imports'
// ...
</script>
```

## Reittien lokalisointi

Nuxt I18n luo lokalisoidut reitit jokaiselle kielelle. Yksinkertaisimmassa asetuksessa tämä tarkoittaa, että jokaiselle reitille luodaan etuliitteellinen versio kielikoodilla.

Kun linkität sovelluksesi sisäisiin reitteihin, sinun täytyy hakea lokalisoitu reitti nykyiselle kielelle. Tämä tehdään Nuxt I18n:n tarjoamilla apufunktioilla.

### Lokalisoidun reitin selvittäminen `$localePath`-funktiolla

`$localePath`-funktiota käytetään hakemaan lokalisoitu reitti tietylle reitille. `<template>`-osion ulkopuoliseen käyttöön funktio palautetaan `useLocalePath`-composablesta.

Funktio hyväksyy kaksi parametria:

- `route`: reitin nimi tai reittiobjekti, jolla on name-ominaisuus
- `locale`: kielikoodi, johon reitti lokalisoidaan — oletuksena nykyinen kieli

::code-group

```vue [page.vue (globaali funktio)]
<template>
  <NuxtLink :to="$localePath('index')">{{ $t('home') }}</NuxtLink>
  <NuxtLink :to="$localePath('index', 'en')">Etusivu englanniksi</NuxtLink>
  <NuxtLink :to="$localePath('user-profile')">Reitti: {{ $t('profile') }}</NuxtLink>
  <NuxtLink :to="$localePath({ name: 'category-slug', params: { slug: category.slug } })">
    {{ category.title }}
  </NuxtLink>
</template>
```

```vue [page.vue (composable)]
<script setup>
const localePath = useLocalePath()
</script>

<template>
  <NuxtLink :to="localePath('index')">{{ $t('home') }}</NuxtLink>
  <NuxtLink :to="localePath('index', 'en')">Etusivu englanniksi</NuxtLink>
  <NuxtLink :to="localePath('user-profile')">Reitti: {{ $t('profile') }}</NuxtLink>
  <NuxtLink :to="localePath({ name: 'category-slug', params: { slug: category.slug } })">
    {{ category.title }}
  </NuxtLink>
</template>
```

::

Koska lokalisoidut reitit voivat muuttua konfiguraatiosi mukaan, reittinimien käyttäminen varmistaa tarkan selvityksen. Nuxt I18n luo tyypit tätä varten, tarjoten tyyppiturvallisen ja paremman kehittäjäkokemuksen. Ottaaksesi nämä tyypit käyttöön, ota `typedPages` käyttöön Nuxt-konfiguraatiossasi.

Reitin nimi vastaa nimiä, jotka Nuxt luo jäsentäessään `pages`-hakemistoasi. Lisätietoja [Nuxt-dokumentaatiosta](https://nuxt.com/docs/guide/directory-structure/pages).

### Kielten välillä vaihtaminen

`$switchLocalePath`-funktio palauttaa lokalisoidun version nykyisen sivun reitistä. Se hyväksyy kielikoodin, johon nykyinen reitti lokalisoidaan.

::code-group

```vue [page.vue (globaali funktio)]
<template>
  <NuxtLink :to="$switchLocalePath('en')">English</NuxtLink>
  <NuxtLink :to="$switchLocalePath('nl')">Nederlands</NuxtLink>
</template>
```

```vue [page.vue (composable)]
<script setup>
const switchLocalePath = useSwitchLocalePath()
</script>

<template>
  <NuxtLink :to="switchLocalePath('en')">English</NuxtLink>
  <NuxtLink :to="switchLocalePath('nl')">Nederlands</NuxtLink>
</template>
```

::

### URL-polku reittiobjektilla

Voit lokalisoida edistyneitä URL-polkuja käyttämällä `useLocaleRoute`-composablea. Tämä on hyödyllistä, kun haluat hallita sisäisiä linkkejä ohjelmallisesti.

`useLocaleRoute` on composable-funktio, joka palauttaa `Route`-objektin tietylle sivulle.

Se toimii kuten `useLocalePath`, mutta palauttaa Vue Routerin selvittämän reittiobjektin täyden reittopolun sijaan. Tämä voi olla hyödyllistä, sillä `useLocalePath`:n palauttama polku ei välttämättä sisällä kaikkea annetun syötteen tietoa (esimerkiksi reittiparametrit, joita sivu ei määrittele).

```vue
<script setup>
const localeRoute = useLocaleRoute()
function onClick() {
  const route = localeRoute({ name: 'user-profile', query: { foo: '1' } })
  if (route) {
    return navigateTo(route.fullPath)
  }
}
</script>

<template>
  <button @click="onClick">Näytä profiili</button>
</template>
```
