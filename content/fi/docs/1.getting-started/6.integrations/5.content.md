## Asennus

Aluksi voit seurata virallista [ohjetta](https://content.nuxt.com/docs/getting-started/installation) tai lyhyesti:

::code-group{sync="pm"}

```bash [pnpm]
pnpm add @nuxt/content
```

```bash [yarn]
yarn add @nuxt/content
```

```bash [npm]
npm install @nuxt/content
```

```bash [bun]
bun add @nuxt/content
```

::

Sitten lisää `@nuxt/content`-moduuli `nuxt.config.ts`-tiedostoon:

```ts [nuxt.config.ts] {4}
export default defineNuxtConfig({
  modules: [
    '@nuxt/ui',
    '@nuxt/content'
  ],
  css: ['~/assets/css/main.css']
})
```

::caution
`@nuxt/content` tulee olla `@nuxt/ui` jälkeen `modules`-listassa, muutoin prose-komponentit eivät ole saatavilla.
::

## Kokoonpano

Käytettäessä Tailwind CSS-luokkia markdown-tiedostoissa tulee varmistaa, että Tailwind voi huomata ja luoda vaaditut hyötyluokat

Oletuksena Tailwindin automaattinen sisällön havaitseminen ei välttämättä huomaa markdown-tiedostoissa olevia luokkia.

Korjauksena voidaa käyttää [`@source`-direktiiviä](https://tailwindcss.com/docs/functions-and-directives#source-directive) CSS-tiedostossa määrittäen sisältökansion:

```css [app/assets/css/main.css]
@import "tailwindcss";
@import "@nuxt/ui";

@source "../../../content/**/*";
```
Tämän ansiosta:
- Tailwind skannaa kaikki markdown-tiedostot sisältökansiosta
- Kaikki markdownissa käytetyt hyötyluokat (kuten `text-primary`) on mukana lopullisessa CSS:ssä
- Dynaamiset luokat MDC- tai omissa Vue-komponenteissa toimivat oikein

::tip
Voit muös käyttää yleiskuvioita ollen tarkempi siinä, mitkä tiedostot skannataan:
- `@source "../../../content/docs/**/*.md"` - Vain markdown docs-kansiossa
- `@source "../../../content/**/*.{md,yml}"` - Sekä markdown että YAML-tiedostot
::

::note{to="https://tailwindcss.com/docs/detecting-classes-in-source-files" target="_blank"}
Lue lisää Tailwindin automaattisesta sisällönhausta ja parhaista menettelytavoista parantaen suorituskykyä.
::

## Komponentit

Voit käyttää `@nuxt/content`-moduulia dokumentaation luomiseen. Sen helpottamiseksi on tehty monia käteviä komponentteja:

- Täysi tekstihaku komennoilla [ContentSearch](https://ui.nuxt.com/docs/components/content-search), korvaten mm. Algolia DocSearch:n
- Navigointipuu [ContentNavigation](https://ui.nuxt.com/docs/components/content-navigation)-komponentilla
- Leijuva Sisällysuettelo [ContentToc](https://ui.nuxt.com/docs/components/content-toc)-komponentilla
- Seuraava / edellinen sivu [ContentSurround](https://ui.nuxt.com/docs/components/content-surround)-komponentilla

## Typografia

Virtauma tarjoaa omanlaisia prose-komponentteja `@nuxt/content`-moduulin avulla. Tämä varmistaa yhtenäiset tyylit, täyden kirjaisimien hallinnan sekä yhteyden Virtauma-järjestelmään, jolloin sisältö näyttää yhtenäiseltä.

::note{to="https://ui.nuxt.com/docs/typography"}
Tutki koko **Typografia**-järjestelmää ja katsele prose-komponentteja rikkaan ja johdonmukaisen sisällön esittelyyn.
::

## Apuohjelmat

### `mapContentNavigation`

Tämä koostaa navigaation `queryCollectionNavigation`:sta ja muuntaa sen rekursiivisesti joukoksi kohteita, joita voidaan käyttää useissa komponenteissa.

`mapContentNavigation(navigation, options?)`
- `navigation`: Navigointipuu (array of ContentNavigationItem).
- `options` (optional):
  - `labelAttribute`: (string) Mitä kenttää käytetään (`title` oletuksena)
  - `deep`: (number or undefined) Määrittää montako navigoinnin tasoa käytetään(`undefined` oletuksena : sisältää kaikki tasot)

**Esim:** Ylemmän esimerkin mukaan, sitä käytetään yleisesti muuntamaan navigointidatan oikeaan muotoon.

```vue [app.vue]
<script setup lang="ts">
import { mapContentNavigation } from '@nuxt/ui/utils/content'
import { findPageBreadcrumb } from '@nuxt/content/utils'

const { data: navigation } = await useAsyncData('navigation', () => queryCollectionNavigation('content'))

const breadcrumb = computed(() => mapContentNavigation(findPageBreadcrumb(navigation?.value, page.value?.path, { indexAsChild: true })).map(({ icon, ...link }) => link), { deep: 0 })
</script>

<template>
  <UPage>
    <UPageHeader v-bind="page">
      <template #headline>
        <UBreadcrumb :items="breadcrumb" />
      </template>
    </UPageHeader>
  </UPage>
</template>
```
