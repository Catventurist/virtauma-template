## Nuxt App

Virtauma pohjautuu alustaan **Nuxt 4**, joten projekti on samalla myös täysi Nuxt-ohjelmisto. Kun luot projektin käyttämällä **Virtauma CLI**:ä, lisäät tason kaikilla Nuxtin herkuilla.

Oletuksena aloituspaketti sisältää vain `content/` ja `public/`-kansion sekä `package.json`:n. Tässä on kaikki tarvittava dokumentaation luomiseen. Voit laajentaa käyttämällä mitä tahansa Nuxt-ominaisuutta, aina from [nuxt.config.ts](https://nuxt.com/docs/guide/directory-structure/nuxt-config):sta [komponentteihin](https://nuxt.com/docs/guide/directory-structure/nuxt-config) tai [lisäosiin](https://nuxt.com/docs/guide/directory-structure/plugins).

::note
Voit käyttää Nuxt 4:n [uutta hakemistorakennetta](https://nuxt.com/docs/getting-started/upgrade#new-directory-structure). Kaikki sovelluksen tiedostot liittyen ulkoasuun lisätään kansioon `app/` selkeämmän rakenteen ja paremman IDE-tehokkuuden varmistamiseksi.
::

## Nuxt-moduulit

Haluatko parantaa sisältöä omilla toiminnoilla? Voit asentaa ja muokata [Nuxt-moduuleja](https://nuxt.com/modules) kuten missä tahansa Nuxt-ohjelmistossa.

Lisätäksesi [Plausible-analytiiikan](https://github.com/nuxt-modules/plausible?utm_source=nuxt.com\&utm_medium=aside-module\&utm_campaign=nuxt.com):

::steps
### Käytä komentoa

```bash [Terminal]
npm install @nuxtjs/plausible
```

### Lisää moduuli `nuxt.config.ts`-tiedostoon

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['@nuxtjs/plausible'],
})
```
::

## Omat komponentit

**Nuxt Content** ja **Nuxt UI** **MDC**-syntaksin kautta mahdollistaa [Nuxt-komponenttien](/fi/docs/essentials/components) lisäämisen Markdowniin vähäisellä vaivalla.

Et ole kiinni vain esiasetetuissa komponenteissa. Sovelluksten avulla on helppoa luoda omia Vue-komponentteja Nuxtiin ja käyttää niitä sisällössä.

Tämä mahdollistaa dynaamiset dokumentoinnit Nuxt-komponenteissa Markdownilla.

## Vue-sivut

Markdown-sivujen lisäksi voit luoda myös Vue-sivuja `pages/`-kansioon.

```vue [pages/hello.vue]
<template>
  <div>
    <h1>Moi</h1>
  </div>
</template>
```

You can also use the `definePageMeta` function to set the page meta, such as using the `default` or `docs` layout, but also to define if the page should display the header and the footer:

```vue [pages/hello.vue]
<script setup lang="ts">
definePageMeta({
  layout: 'default',
  // Poista yläpalkki
  header: false,
  // Poista alapalkki
  footer: false,
})
</script>
```

## Omat asettelut

Virtauma käyttää oletuksena kahta asettelua:
- `default`-asettelu etusivulle ja omille Vue-sivuille
- `docs`-näkymä sopii dokumentaatiosivuille

Käyttääksesi muita asetteluja voit luoda omasi `app/layouts/`-kansioon.

```vue [app/layouts/custom.vue]
<template>
  <main class="oma-asettelu">
    <slot />
  </main>
</template>
```
