Virtauma tarjoaa **natiivin käännöstuen** perustuen `@nuxtjs/i18n`-moduuliin, jonka avulla sisältöa voidaan luoda useammalla kielellä automaattisilla reitityksillä ja sisällönhallinnalla.

## Ominaisuudet

- **Sisäänrakennettuna i18n-moduli**: Natiivi integraatio käyttäen `@nuxtjs/i18n`:a
- **Dynaaminen kielen reititys**: Automaattiset URL-prefiksit kielikoodeilla (`/en/docs`, `/fi/docs`)
- **Kielikohtaiset sisältökokoelmat**: Kieliriippuvainen sisällönhallinta
- **Kielen vaihtaja**: Sisäänrakennettu kielen vaihtamisen komponentti
- **Yksikieliset asetukset**: Helpot kieliasetukset yhdelle kielelle tiedostossa `app.config.ts`

## Yksikieliset asetukset

Jos teet sisältöä vain yhdelle kielelle (ilman `@nuxtjs/i18n`-moduulia) voit määrittää kielen tiedostossa `app.config.ts`. Tämä on kätevää ulkoasukomponenttien ja sanaston määrittelemisessä.

```ts [app.config.ts]
export default defineAppConfig({
  app: {
    locale: 'fi', // Aseta kieli (oletus: 'en')
  }
})
```

## Monikielisyys

Monikieliselle sisällölle käytetään `@nuxtjs/i18n`-integraatiota.

### Asenna toimivaan kirjastoon

Lisätäksesi i18n:m Virtauma-projektiin lisää `@nuxtjs/i18n`-moduuli `nuxt.config.ts`-tiedostoon ja määritä kielet:

```typescript [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['@nuxtjs/i18n'],
  i18n: {
    defaultLocale: 'en',
    locales: [
      { code: 'en', name: 'English' },
      { code: 'fi', name: 'Suomi' },
    ],
  }
})
```

::warning
Virtauma muuttaa `@nuxtjs/i18n`-strategian `prefix`:si.
::

## Uusi projekti i18n-mallipohjalla

Luodessa uutta projektia voit valita i18n-mallipohjan esimääritetyllä monikielisyydellä:

```bash [Terminal]
npm create nuxt@latest -t i18n
```

## Hakemistorakenne

Kun i18n on käytössä, sisältö lisätään kielen perusteella `content/`-kansioon:

```bash
content/
├── en/                   # Sisältö englanniksi
│   ├── index.md          # Etusivu englanniksi
│   ├── getting-started/
│   │   ├── installation.md
│   │   └── configuration.md
│   └── guide/
│       └── advanced.md
├── fi/                   # Sisältö suomeksi
│   ├── index.md          # Etusivu suomeksi
│   ├── getting-started/
│   │   ├── installation.md
│   │   └── configuration.md
│   └── guide/
│       └── advanced.md
```

::warning
Jokaisen kielen tulee heijastaa samaa hakemistorakennetta, jotta navigaatio on johdonmukaista kielten välillä.
::

## Kieliasetuksen varavaihtoehto

Virtauma varoittaa ja ohittaa kielet, joita ei löydy `content/`-kansiosta. Puuttuvia kieliä ei rekisteröidä.

::tip
Tämä auttaa kun ohitetaan Virtauma ja käytetään `@nuxtjs/i18n` sivustossa, mutta halutaan sisältö vain kielten osajoukossa.
::

Virtauma havaitsee kielet Nuxt-asetuksista:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['@nuxtjs/i18n'],
  i18n: {
    locales: ['en', 'fi', 'es'],
    defaultLocale: 'en'
  }
})
```
Mutta rekisteröi vain perustuen `content/`-kansion rakenteeseen:

```bash
content/
├─ en/           # rekisteröity ✅
├─ fi/           # rekisteröity ✅ (jos löytyy)
└─ fr/           # ohitettu 🚫 (jos puuttuu)
```
Jos käyttäjä löytää sivun josta puuttuu kieli, Virtauma palauttaa **oletuskielen**.

::warning
`defaultLocale` tulee määrittää i18n-asetuksissa ja varmistaa että sisältöä löytyy kansiosta `content/<defaultLocale>`.
::
